{\footnotesize\ttfamily uint8\+\_\+t Conway\+Bromage\+::successors (\begin{DoxyParamCaption}\item[{uint64\+\_\+t}]{Kmer }\end{DoxyParamCaption}) const}



Returns the successors of a (k-\/1)-\/mer. 

The (k-\/1)-\/mer can either be canonical or not. A (k-\/1)-\/mer has at minimum 0 successors and maximum 8 successors. The function doesn\textquotesingle{}t check if the (k-\/1)-\/mer is present. \hypertarget{class_conway_bromage_autotoc_md6}{}\doxyparagraph{How it works ?}\label{class_conway_bromage_autotoc_md6}
Let\textquotesingle{}s say the function takes as a parameter the integer which represents the (k-\/1)-\/mer G\+TT and we assume we are in A\+C\+GT encoding. First, the method will generate the following k-\/mers \+: G\+T\+TA, G\+T\+TC, G\+T\+TG, G\+T\+TT, A\+G\+TT, C\+G\+TT, G\+G\+TT, T\+G\+TT. Then, it will check if the canonical version of these k-\/mers are present. Let\textquotesingle{}s suppose that only the canonical version of these k-\/mers are present \+: G\+T\+TA, G\+T\+TT, C\+G\+TT, T\+G\+TT. G\+T\+TA and G\+T\+TT are, what we call next k-\/mers so we will store the information in the 4 left bits of the result (uint8\+\_\+t). C\+G\+TT and T\+G\+TT are previous k-\/mers so, this time, the information will be stored in the 4 right bits of the result. Thus, the function will return 1001 0101 which corresponds to 149 in base 10. It means that, in this example, the successors of the (k-\/1)-\/mer G\+TT are T\+TA, T\+TT, C\+GT and T\+GT.
